### 1.2 技术栈迁移路径

#### 阶段 1: 从 JavaScript 到 C++

**为什么必须迁移到 C++？**

| 需求 | JavaScript/Node.js | C++ | 选择 |
|------|-------------------|-----|------|
| **嵌入式 Linux 支持** | ⚠️ 可行但性能差 | ✅ 原生支持 | C++ |
| **实时性能保证** | ❌ 受 V8 GC 影响 | ✅ 无 GC，确定性延迟 | C++ |
| **硬件直接访问** | ❌ 需要 FFI | ✅ 原生硬件访问 | C++ |
| **功耗优化** | ❌ 高功耗 | ✅ 极致优化 | C++ |
| **启动时间** | ❌ 慢（需加载 V8） | ✅ 快（<1秒） | C++ |

**迁移工具**: JUCE Framework

[JUCE](https://juce.com/) 是音频应用的事实标准 C++ 框架：
- ✅ **跨平台** - Linux、Windows、macOS、iOS、Android、嵌入式
- ✅ **完整的音频栈** - MIDI、合成器、效果器、VST 插件
- ✅ **Web Audio 移植** - [JUCE WebAssembly 版本](https://github.com/jariseon/JUCE) 已存在
- ✅ **商业支持** - Ableton、Native Instruments 都在用
- ✅ **嵌入式优化** - [Elk Audio 特殊版本](https://github.com/elk-audio/JUCE) 支持超低延迟

**迁移示例**（你的 YIN 算法）:

```javascript
// 当前 JavaScript 实现 (js/pitch-detector.js)
class YINDetector {
    detectPitch(buffer) {
        const yinBuffer = this._difference(buffer);
        const tau = this._absoluteThreshold(yinBuffer);
        return this._sampleRate / tau;
    }
}
```

```cpp
// 迁移后 C++ 实现 (JUCE)
class YINDetector {
public:
    float detectPitch(const juce::AudioBuffer<float>& buffer) {
        auto yinBuffer = difference(buffer);
        auto tau = absoluteThreshold(yinBuffer);
        return sampleRate / tau;
    }
};
```

**复用率**: 90% 的算法逻辑可直接翻译，主要区别在内存管理。

#### 阶段 2: 神经网络模型迁移

**当前**: Web 原型使用传统 DSP（YIN + FFT）

**硬件**: 部署 RAVE 神经音频合成 [^1]

**迁移路径**:

```
Python (训练)               C++ (推理)
    ↓                          ↓
PyTorch RAVE 模型 → ONNX → ONNX Runtime (C++)
    ↓                          ↓
训练数据集           → 嵌入式 Linux (Jetson/RPi)
    ↓                          ↓
.onnx 文件          → 实时音频合成 (<10ms)
```

**已验证的方案** [^1]:
- RAVE 模型在 **Jetson Nano** 上实时运行（48kHz, 20x faster than real-time）
- Neurorack 项目已实现 EuroRack 格式的 RAVE 硬件合成器
- 472 GFLOPS 的并行计算能力（Jetson Nano $99）

**你的优势**: Web 原型已经验证了音高检测和合成逻辑，迁移到 RAVE 只需替换合成引擎。

### 1.3 推荐的技术栈（硬件产品）

**最终技术栈**:

```
硬件平台: NVIDIA Jetson Orin Nano (高端) / Raspberry Pi 5 (中端)
    ↓
操作系统: Embedded Linux (Yocto / Ubuntu Core)
    ↓
实时内核: PREEMPT_RT Patch (延迟 <5ms)
    ↓
音频框架: JUCE 7.x + ALSA/JACK
    ↓
DSP 引擎
├── 音高检测: FCPE (神经网络) 或 YIN (传统)
├── 音频合成: RAVE (神经) 或 JUCE 合成器 (传统)
└── 效果器: JUCE FX 链
    ↓
UI 层: Qt 6 (触摸屏) / LVGL (小屏幕)
    ↓
硬件接口
├── I2S DAC (音频输出)
├── I2S ADC (麦克风输入)
├── GPIO (按钮/旋钮)
└── SPI/I2C (显示屏)
```

**与 Web 原型对比**:

| 层次 | Web 原型 | 硬件产品 | 迁移复杂度 |
|------|---------|---------|-----------|
| **DSP 核心** | JavaScript YIN/FFT | C++ YIN/FCPE | 低（算法相同） |
| **音频合成** | Tone.js | JUCE/RAVE | 中（框架不同） |
| **音频 I/O** | Web Audio API | ALSA/JACK | 中（硬件适配） |
| **UI 渲染** | DOM/Canvas | Qt/LVGL | 高（完全重写） |
| **状态管理** | JavaScript Store | C++ State Machine | 低（逻辑相同） |

## 第二部分：硬件平台选型深度分析

### 2.1 三大平台对比

根据你的需求（实时音频 + 神经网络 + 量产），以下是详细对比：

#### Option A: NVIDIA Jetson Orin Nano（推荐高端版）

**规格**:
- CPU: 6核 ARM Cortex-A78 @ 1.5GHz
- GPU: 1024核 NVIDIA Ampere
- AI: 40 TOPS (INT8)
- 内存: 4GB/8GB LPDDR5
- 价格: $199 (4GB) / $299 (8GB)

**优势**:
- ✅ **RAVE 神经合成完美支持** [^1] - TensorRT 加速，<10ms 延迟
- ✅ **未来扩展性** - 可运行复杂的 AI 模型（FCPE、CREPE）
- ✅ **成熟生态** - NVIDIA 官方支持，大量开源项目
- ✅ **专业音频案例** - Neurorack 已验证可行性 [^1]

**劣势**:
- ❌ **功耗较高** - 5-15W（需散热）
- ❌ **成本** - 是 Raspberry Pi 的 3-4 倍
- ❌ **供应链风险** - 依赖单一厂商（NVIDIA）

**适用场景**: **专业级产品**（售价 $500-$1000+）

**你的项目匹配度**: ⭐⭐⭐⭐⭐ (完美匹配，支持 RAVE 是关键优势)

---

#### Option B: Raspberry Pi 5（推荐中端版）

**规格**:
- CPU: 4核 ARM Cortex-A76 @ 2.4GHz
- GPU: VideoCore VII
- 内存: 4GB/8GB LPDDR4X
- 价格: $60 (4GB) / $80 (8GB)

**优势**:
- ✅ **成本低** - 是 Jetson 的 1/3-1/4
- ✅ **供应链稳定** - 全球供应，易于采购
- ✅ **社区庞大** - 海量教程和开源项目
- ✅ **音频优化** - [Elk Audio OS](https://elk.audio/) 专为低延迟音频设计
- ✅ **DDSP 支持** - CPU 足够运行 DDSP（简化版神经合成）

**劣势**:
- ⚠️ **RAVE 性能有限** - 无法运行完整 RAVE（可运行 DDSP）
- ⚠️ **散热需求** - 需主动散热（风扇）
- ⚠️ **音频延迟** - 比 Jetson 略高（15-30ms vs <10ms）

**适用场景**: **消费级产品**（售价 $200-$500）

**你的项目匹配度**: ⭐⭐⭐⭐ (性价比高，适合初期量产)

---

#### Option C: ESP32-S3 / STM32H7（不推荐主控）

**规格** (ESP32-S3):
- CPU: 双核 Xtensa LX7 @ 240MHz
- 内存: 512KB SRAM
- 价格: $2-$5

**优势**:
- ✅ **成本极低** - 大规模量产成本 <$5
- ✅ **功耗极低** - <1W
- ✅ **集成度高** - WiFi/BLE 内置

**劣势**:
- ❌ **算力不足** - 无法运行神经网络
- ❌ **内存太小** - 无法运行复杂 DSP
- ❌ **音频质量** - ADC 性能差 [^2]

**适用场景**: **MIDI 控制器**（发送数据到电脑/合成器）

**你的项目匹配度**: ⭐ (仅适合作为辅助控制器，不适合主控)

**策略**: 同时开发两个版本，覆盖不同市场

| 产品线 | 平台 | 定位 | 价格 | 功能 |

| **PCB** | 5-10 片 | 小批量易获得 |

**你的产品 MOQ 策略**:

**阶段 1: 原型**（10-50 台）
- PCB: 深圳嘉立创（5 片起）
- 外壳: 3D 打印（$50/个）
- 组装: 手工/小作坊
- **成本**: $500-800/台

**阶段 2: 小批量**（100-500 台）
- PCB: 批量打样（降至 $10/片）

### 7.3 与 TypeScript 报告的关联

**之前的 TypeScript 分析结论**: Web 原型应保持 JavaScript + 宽松 TypeScript

**硬件产品的影响**: **完全正确，且更加重要**

**原因**:
1. **Web 原型是硬件的"数字孪生"**
   - 快速测试新算法（无需等硬件）
   - 用户反馈迭代快（远程更新）
   - 市场验证工具（转化率追踪）

2. **代码复用策略**
   ```
   Web 原型（JavaScript）
       ↓
   算法验证 + 用户测试
       ↓
   迁移到 JUCE（C++）  ← 手工翻译（90% 相同逻辑）
       ↓
   硬件产品固件
   ```

3. **长期维护**
   - Web 版: 免费版/试用版
   - 硬件版: 付费产品
   - 共享研发成果（算法改进同时应用）

**结论**: **不要放弃 Web 版，它是硬件产品的最佳营销工具和研发平台。**
